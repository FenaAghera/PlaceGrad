Python Interview Questions

Top 20 questions to prepare for your Python interview.

1. What is Python?

▼

Python is a high-level, interpreted, interactive, and object-oriented scripting language. It is a very popular language for web development, data science, machine learning, and automation.

2. What is PEP 8?

▼

PEP 8 is a style guide for Python code. It provides conventions for writing clean and readable code, such as indentation, naming conventions, and line length. Following PEP 8 makes your code more consistent and easier for others to read.

3. What are the key features of Python?

▼

Python's key features include its simplicity, readability, portability, object-oriented and procedural programming support, extensive standard library, and a vast ecosystem of third-party libraries.

4. What is the difference between a list and a tuple?

▼

A list is a mutable, ordered sequence of elements, enclosed in square brackets []. A tuple is an immutable, ordered sequence, enclosed in parentheses (). Once a tuple is created, its elements cannot be changed, added, or removed.

5. Explain the GIL (Global Interpreter Lock).

▼

The Global Interpreter Lock (GIL) is a mutex that protects access to Python objects, preventing multiple native threads from executing Python bytecodes at once. This means that while Python's `threading` module can run multiple threads, only one thread can be executing Python code at any given time. The GIL is a major reason why Python is not ideal for CPU-bound tasks in a multi-threaded environment, but it does not affect I/O-bound tasks.

6. What are decorators in Python?

▼

A decorator is a design pattern in Python that allows you to add new functionality to an existing function or class without modifying its structure. Decorators are essentially functions that take another function as an argument and return a new function that usually extends the behavior of the original function.

7. Explain `__init__` in Python.

▼

The __init__ method is a special method in Python classes, known as a constructor. It is automatically called when a new object of a class is created. It is used to initialize the state of the object, for example, by setting initial values for instance variables.

8. What is a generator in Python?

▼

A generator is a special type of function that returns an iterator, allowing you to iterate over a sequence of values one at a time. Generators use the yield keyword instead of return. They are memory-efficient because they do not store all the values in memory at once, generating them on the fly as needed.

9. How is memory managed in Python?

▼

Python's memory management is handled automatically by the Python Memory Manager. The memory is allocated in a private heap, and a garbage collector is used to automatically reclaim memory from objects that are no longer in use.

10. What is a lambda function?

▼

A lambda function is a small, anonymous function defined with the lambda keyword. It can take any number of arguments but can only have one expression. They are typically used for simple, short-term operations where a full function definition is not necessary.

11. Explain the difference between `range()` and `xrange()` in Python 2.

▼

In Python 2, range() returns a list of numbers, which can consume a lot of memory for large ranges. xrange(), on the other hand, returns a generator object that generates numbers on the fly, making it more memory-efficient. In Python 3, range() now behaves like xrange() from Python 2, returning a generator-like object, and xrange() no longer exists.

12. What is the difference between `.py` and `.pyc` files?

▼

A .py file contains the source code of a Python program. A .pyc file is a compiled bytecode file that is created when the Python interpreter executes a .py file. This bytecode is a low-level representation of the source code that can be executed more quickly on subsequent runs.

13. What is a dictionary in Python?

▼

A dictionary is an unordered collection of data values used to store data in key-value pairs. Dictionaries are optimized for retrieving values when the key is known. They are mutable and are defined with curly braces {}.

14. How do you handle exceptions in Python?

▼

Exceptions in Python are handled using a try-except block. The code that might raise an error is placed in the try block, and the code to handle the error is placed in the except block. This prevents the program from crashing and allows for graceful error handling.

15. What is the difference between shallow and deep copy?

▼

A shallow copy creates a new compound object and inserts references to the objects found in the original. It only duplicates the top-level structure, so changes to nested objects are reflected in both copies. A deep copy creates a completely new independent compound object and recursively copies all objects found in the original, so changes to nested objects do not affect the original.

16. What is the difference between `is` and `==`?

▼

The == operator compares the values of two objects to see if they are equal. The is operator compares the identity of two objects to see if they are the exact same object in memory.

17. What is a virtual environment?

▼

A virtual environment is a tool that keeps the dependencies required by different projects in separate locations, preventing conflicts. It creates an isolated environment for each project, ensuring that libraries installed for one project do not affect others.

18. What is the purpose of `self` in a class?

▼

The `self` parameter is a reference to the instance of the class. It is used to access variables and methods belonging to the class instance. It is the first argument to any method in a class and must be included, though you can rename it.

19. How do you write comments in Python?

▼

Single-line comments are created with the `#` symbol. Multi-line comments can be created by placing the text inside triple quotes (either single '''...''' or double """..."""). These are technically multi-line strings, but they are often used as comments.

20. What is a docstring?

▼

A docstring (documentation string) is a string literal that occurs as the first statement in a module, function, class, or method definition. It is a way of documenting the code's purpose and functionality. Docstrings are accessible at runtime via the __doc__ attribute.
