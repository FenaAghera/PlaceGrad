Java Interview Questions

Top 20 questions to prepare for your Java interview.

1. What is Java?

▼

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible. It is a general-purpose language intended to let application developers write once, run anywhere (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation.

2. What is JVM, JRE, and JDK?

▼

JVM (Java Virtual Machine) is an abstract machine that provides a runtime environment in which Java bytecode can be executed. JRE (Java Runtime Environment) is the software package that contains the JVM, class libraries, and other files required to run a Java application. JDK (Java Development Kit) is a superset of JRE, containing everything in JRE plus tools for developing Java applications, such as a compiler (javac), debugger, and archiver.

3. What are the four pillars of OOP in Java?

▼

The four pillars of OOP are: Encapsulation (bundling data and methods into a single unit), Inheritance (allowing a class to inherit properties from another class), Polymorphism (the ability of an object to take on many forms), and Abstraction (hiding complex implementation details from the user).

4. What is the difference between `==` and `.equals()`?

▼

The `==` operator is used to compare object references (memory addresses) to see if they point to the same object. The `.equals()` method is used to compare the actual content or value of two objects. For primitive data types, `==` is used for value comparison, but for objects, `.equals()` is the correct way to compare their values.

5. What is method overloading and overriding?

▼

Method overloading occurs when a class has multiple methods with the same name but different parameters (number of arguments or their data types). It is a compile-time polymorphism. Method overriding occurs when a subclass provides a specific implementation for a method that is already defined in its superclass. It is a runtime polymorphism.

6. Explain the concept of a constructor.

▼

A constructor is a special type of method that is used to initialize the state of an object. It is called automatically when an object is created using the `new` keyword. A constructor has the same name as its class and does not have a return type. It can be overloaded to provide multiple ways to create an object.

7. What is a static keyword in Java?

▼

The `static` keyword can be applied to variables, methods, blocks, and nested classes. A static member belongs to the class itself rather than to any specific instance of the class. This means you can access static members without creating an object of the class.

8. What is the difference between an abstract class and an interface?

▼

An **abstract class** can have both abstract and non-abstract methods, can have a constructor, and can contain instance variables. A class can extend only one abstract class. An **interface** can only have abstract methods (in Java 8 and later, it can have default and static methods), cannot have a constructor, and only has static and final variables. A class can implement multiple interfaces.

9. What is the final keyword?

▼

The `final` keyword is used to restrict the user. A `final` variable's value cannot be changed. A `final` method cannot be overridden by subclasses. A `final` class cannot be inherited by other classes.

10. What is a checked vs. unchecked exception?

▼

A **checked exception** is an exception that is checked at compile-time. The compiler forces the programmer to handle it, either by using a `try-catch` block or by declaring it in the method signature with `throws`. Examples include `IOException` and `SQLException`. An **unchecked exception** is not checked at compile-time and is typically caused by programming errors. Examples include `NullPointerException` and `ArrayIndexOutOfBoundsException`.

11. What is garbage collection in Java?

▼

Garbage collection is an automatic memory management process in Java. The Garbage Collector automatically identifies and reclaims memory from objects that are no longer referenced by the program. This frees the programmer from manual memory deallocation, which helps prevent memory leaks.

12. What are the differences between an Array and an ArrayList?

▼

An **Array** is a fixed-size, ordered collection of elements of the same data type. Once created, its size cannot be changed. An **ArrayList** is a resizable array implementation of the `List` interface. Its size can grow dynamically as elements are added or removed.

13. What is a package in Java?

▼

A package is a mechanism in Java for organizing classes, interfaces, and subpackages. It helps in modularizing the codebase, preventing naming conflicts, and providing access control.

14. Explain the `this` keyword.

▼

The `this` keyword is a reference variable that refers to the current object. It can be used to differentiate between instance variables and local variables with the same name, to invoke the current class's constructor, or to pass the current object as an argument.

15. What are primitive and non-primitive data types?

▼

**Primitive data types** are predefined and are not objects. They include `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, and `char`. **Non-primitive** (or reference) data types are objects created by the programmer or provided by the Java library. They include `String`, `Array`, and classes.

16. What is a constructor chaining?

▼

Constructor chaining is the process of calling one constructor from another constructor. This is useful when you have multiple constructors and want to avoid redundant code. You can use `this()` to call a constructor in the same class or `super()` to call a constructor of the parent class.

17. What is a `try-with-resources` statement?

▼

The `try-with-resources` statement is a feature introduced in Java 7 that ensures that each resource (such as a file, socket, or database connection) is automatically closed after the `try` block is executed, regardless of whether the block completes normally or throws an exception. This helps prevent resource leaks.

18. What is the `String` pool in Java?

▼

The String pool is a special memory area in the heap where Java stores literal strings. When a new string literal is created, the JVM first checks the String pool. If an identical string already exists, it returns a reference to that string. If not, it creates a new string and adds it to the pool. This improves memory efficiency.

19. What is the difference between a `LinkedList` and an `ArrayList`?

▼

An `ArrayList` is an array-based implementation of the `List` interface. It provides fast random access but is slow for insertions and deletions in the middle of the list. A `LinkedList` is a doubly-linked list implementation. It is fast for insertions and deletions but slow for random access because it has to traverse the list from the beginning.

20. What is a `thread` in Java?

▼

A thread is the smallest unit of a process. Java provides a multi-threading feature, which allows multiple threads to run concurrently within a single program. This can be used to perform multiple tasks at the same time, such as a user interface thread and a background data processing thread.
