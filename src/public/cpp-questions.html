C++ Interview Questions

Top 20 questions to prepare for your C++ interview.

1. What is C++?

▼

C++ is a powerful, general-purpose, object-oriented programming language. It is an extension of the C language and is widely used in systems programming, game development, high-performance computing, and embedded systems.

2. What is the difference between C and C++?

▼

C is a procedural language, while C++ is an object-oriented language. C++ supports key OOP concepts like classes, objects, inheritance, polymorphism, and encapsulation, which are not available in C. C++ also has a richer standard library.

3. Explain the four pillars of OOP in C++.

▼

The four pillars are: Encapsulation (binding data and methods into a single unit), Inheritance (creating a new class from an existing one), Polymorphism (the ability of a single function or operator to act on different data types), and Abstraction (hiding complex implementation details).

4. What is the difference between `new` and `malloc()`?

▼

The `new` operator is a C++ keyword that allocates memory for an object and calls its constructor. `malloc()` is a C library function that only allocates raw memory; it does not call the constructor. `new` returns a typed pointer, whereas `malloc()` returns a `void*` pointer that must be cast. Memory allocated with `new` must be freed with `delete`, while `malloc()` requires `free()`.

5. Explain the concept of a pointer in C++.

▼

A pointer is a variable that stores the memory address of another variable. Pointers are used for dynamic memory allocation, to pass large objects to functions efficiently, and for implementing data structures like linked lists.

6. What is a virtual function?

▼

A virtual function is a member function in a base class that you expect to redefine in a derived class. By declaring a function virtual, you enable dynamic polymorphism, where the function to be called is determined at runtime based on the object's type.

7. What is the difference between a class and a struct?

▼

In C++, the main difference is the default access specifier. Members of a `class` are `private` by default, while members of a `struct` are `public` by default. Structs are generally used for simple data-only structures, while classes are used for more complex objects with both data and behavior.

8. What is a destructor?

▼

A destructor is a special member function that is automatically called when an object is destroyed. It is used to deallocate memory and perform any necessary cleanup for the object before it is removed from memory.

9. What is a copy constructor?

▼

A copy constructor is a special constructor that creates a new object as a copy of an existing object. It is called when an object is passed by value to a function or returned by value from a function. It is important for performing a deep copy to prevent issues with shared pointers.

10. What is a smart pointer?

▼

A smart pointer is an object that behaves like a pointer but provides automatic memory management. It is a class that wraps a raw pointer, and when the smart pointer goes out of scope, it automatically deallocates the memory it points to, preventing memory leaks.

11. Explain the `const` keyword.

▼

The `const` keyword is used to specify that a variable's value cannot be changed. It can be used with variables, function arguments, function return types, and member functions to ensure that data is not modified accidentally.

12. What is the Standard Template Library (STL)?

▼

The STL is a set of C++ template classes and functions that provide common data structures and algorithms. It includes containers (like `vector`, `list`, `map`), iterators, algorithms (like `sort`, `find`), and function objects, making it a powerful and reusable library.

13. What is a namespace?

▼

A namespace is a feature that provides a way to group related identifiers (like classes, functions, and variables) to avoid naming conflicts. It creates a declarative region and ensures that names inside the namespace do not clash with names in other parts of the code.

14. What is a friend function and a friend class?

▼

A **friend function** is a non-member function that is granted access to the `private` and `protected` members of a class. A **friend class** is a class whose member functions are all friend functions of another class, giving it access to the other class's private members.

15. What is the difference between `public`, `private`, and `protected`?

▼

These are access specifiers. **`public`** members are accessible from anywhere. **`private`** members are only accessible from within the same class. **`protected`** members are accessible within the same class and by its derived classes.

16. What is operator overloading?

▼

Operator overloading is a C++ feature that allows you to redefine how standard operators (like `+`, `-`, `*`) behave when applied to user-defined data types. This allows for more intuitive and readable code.

17. What is a pure virtual function?

▼

A pure virtual function is a virtual function in a base class that has no implementation in the base class. It is declared by placing `= 0` at the end of the function declaration. A class that contains a pure virtual function is called an abstract class, and it cannot be instantiated.

18. Explain exception handling in C++.

▼

Exception handling is a mechanism to handle runtime errors. It is implemented using `try`, `throw`, and `catch` blocks. The code that might throw an exception is placed in a `try` block, an exception is `throw`n when an error occurs, and it is handled by a `catch` block.

19. What is a constructor and a destructor?

▼

A **constructor** is a special member function that initializes a new object of a class. It is called automatically when an object is created. A **destructor** is a special member function that is called when an object is destroyed. It is used to deallocate resources acquired by the object.

20. What is a static member?

▼

A static member (variable or function) of a class belongs to the class itself rather than to an instance of the class. All instances of the class share a single copy of the static member. Static members can be accessed without creating an object of the class.
